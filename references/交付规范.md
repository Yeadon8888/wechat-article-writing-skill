# 交付规范

一条任务一个交付包。不要默认把所有形态都一起给出。

先判断主链路，再按对应包交付。

## 一、文章包

当主链路是文章时，默认交付：

1. 完整文章正文
2. 3 个标题方向
3. 100 字内摘要或导语
4. 配图/截图建议
5. 结尾 CTA 或评论区互动话题

可选补充：

- 如果主题适合视频化，可额外补一句“可继续改口播”
- 但不要默认直接附整份口播稿

## 二、口播包

当主链路是口播时，默认交付：

1. 一句话卖点
2. 封面文案或视频标题方向
3. 口播稿或口播提纲
4. 2-5 句可做字幕/切片的传播句
5. 收尾 CTA

可选补充：

- 如果需要拍摄执行，再升级成分镜包
- 不要默认补完整文章正文

## 三、分镜包

当用户明确要分镜时，在口播包基础上追加：

1. 视频定位：平台、时长、目标观众、风格
2. 镜头脚本
3. 画面建议
4. 字幕重点
5. 音效/剪辑提示

## 四、双链路联动包

只有当用户明确说“文章和口播都要”或任务是拆条/分发时，才交付：

1. 共享层总结：主题判断、核心承诺、支撑案例
2. 文章包
3. 口播包
4. 共用标题/封面方向
5. 共用金句

注意：

- 文章稿和口播稿分开呈现
- 不要做成一份混合文本
- 两份内容允许共享观点，不允许共享结构

## 五、精修包

当任务是诊断精修时，默认交付：

1. 主要问题定位
2. 修改方向
3. 重写后的关键部分
4. 是否建议整稿重做

## 六、系统沉淀包

当任务是系统沉淀时，默认交付：

1. 要写入哪个目录
2. 要沉淀什么资产
3. 推荐写入格式
4. 是否值得进入方法论
